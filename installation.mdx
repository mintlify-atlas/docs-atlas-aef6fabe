# Installation Guide

This comprehensive guide covers everything you need to install, configure, and deploy Karma POS for both development and production environments.

## System Requirements

### Required Software

Before installing Karma POS, ensure your system meets these requirements:

| Software | Version | Purpose |
|----------|---------|----------|
| Node.js | 24.x or higher | JavaScript runtime |
| npm | 10.x or higher | Package manager |
| Angular CLI | 19.0.6+ | Build tooling |
| Git | Latest | Version control |

<Note>
Karma POS is tested with Node.js v24.13.1 and Angular CLI 19.0.6. Using older versions may cause compatibility issues.
</Note>

### Recommended Environment

- **Operating System**: Linux, macOS, or Windows 10/11
- **RAM**: 4GB minimum, 8GB recommended
- **Disk Space**: 500MB for application + 1GB for dependencies
- **Browser**: Chrome 90+, Firefox 88+, Safari 14+, or Edge 90+

## Installation Steps

### Step 1: Install Node.js

If you don't have Node.js installed:

<CodeGroup>

```bash Linux (using nvm)
# Install nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Install Node.js 24
nvm install 24
nvm use 24

# Verify installation
node --version  # Should show v24.x.x
npm --version   # Should show 10.x.x
```

```bash macOS (using Homebrew)
# Install Homebrew if needed
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install Node.js
brew install node@24

# Verify installation
node --version
npm --version
```

```bash Windows
# Download the installer from https://nodejs.org/
# Run the installer and follow the prompts
# Open a new terminal and verify:
node --version
npm --version
```

</CodeGroup>

### Step 2: Install Angular CLI

Install the Angular CLI globally:

```bash
npm install -g @angular/cli@19
```

Verify the installation:

```bash
ng version
```

You should see Angular CLI version 19.0.6 or higher.

### Step 3: Clone or Extract the Project

If using Git:

```bash
git clone <repository-url>
cd sakai-ng
```

Or extract the project archive to your desired location.

## Configuration

### Application Configuration

The main application configuration is in `src/app.config.ts`:

```typescript
import { ApplicationConfig, LOCALE_ID } from '@angular/core';
import { provideRouter } from '@angular/router';
import { provideHttpClient, withFetch } from '@angular/common/http';
import { provideAnimationsAsync } from '@angular/platform-browser/animations/async';
import { providePrimeNG } from 'primeng/config';
import Aura from '@primeng/themes/aura';

export const appConfig: ApplicationConfig = {
  providers: [
    provideRouter(appRoutes),
    provideHttpClient(withFetch()),
    provideAnimationsAsync(),
    providePrimeNG({
      theme: { 
        preset: Aura, 
        options: { darkModeSelector: '.app-dark' } 
      }
    }),
    { provide: LOCALE_ID, useValue: 'es-CO' }
  ]
};
```

### API Configuration

Configure the backend API URL in `src/app/components/services/pos.service.ts`:

```typescript
private apiUrl = 'https://localhost:7197/api/';
```

<Warning>
Update this URL to point to your actual backend API server before deploying to production.
</Warning>

### Angular Configuration

## Running the Application

### Development Server

Start the development server:

```bash
npm start
```

Or using Angular CLI:

```bash
ng serve
```

The application will be available at `http://localhost:4200/`.

**Development server features:**
- Hot module replacement (HMR)
- Automatic recompilation on file changes
- Source maps for debugging
- Error overlay in browser

<Tip>
Use `ng serve --open` to automatically open the browser when the server starts.
</Tip>

### Watch Mode

For continuous building without serving:

```bash
npm run watch
```

This builds the project in development mode and watches for file changes.

## Building for Production

### Production Build

Create an optimized production build:

```bash
npm run build
```

Or:

```bash
ng build
```

This creates optimized bundles in the `dist/sakai-ng/` directory.

**Production build optimizations:**
- Code minification and uglification
- Tree shaking to remove unused code
- Ahead-of-Time (AOT) compilation
- CSS optimization and purging
- Asset hashing for cache busting

### Build Configurations

The project includes two build configurations:

<CodeGroup>

```bash Production (default)
ng build --configuration production

# Features:
# - Optimized bundle size
# - No source maps
# - AOT compilation
# - Output hashing
```

```bash Development
ng build --configuration development

# Features:
# - Faster builds
# - Source maps enabled
# - No optimization
# - Better debugging
```

</CodeGroup>

### Build Budgets

The production build enforces size budgets:

```json
"budgets": [
  {
    "type": "initial",
    "maximumWarning": "1mb",
    "maximumError": "5mb"
  },
  {
    "type": "anyComponentStyle",
    "maximumWarning": "2kb",
    "maximumError": "4kb"
  }
]
```

<Warning>
If your bundle exceeds these limits, the build will fail. Consider code splitting or lazy loading modules.
</Warning>

## Deployment

### Deploy to Vercel

Karma POS includes Vercel configuration in `vercel.json`:

```bash
# Install Vercel CLI
npm install -g vercel

# Deploy
vercel
```

### Deploy to Other Platforms

After building for production, deploy the `dist/sakai-ng/` directory to:

- **Nginx**: Point document root to `dist/sakai-ng/browser/`
- **Apache**: Configure virtual host to serve from `dist/sakai-ng/browser/`
- **AWS S3**: Upload contents to S3 bucket with static website hosting
- **Firebase Hosting**: Use `firebase deploy`
- **Netlify**: Drag and drop the `dist` folder or connect via Git

### Server Configuration

For Single Page Applications (SPAs), configure your server to redirect all routes to `index.html`:

<CodeGroup>

```nginx Nginx
location / {
  try_files $uri $uri/ /index.html;
}
```

```apache Apache
<IfModule mod_rewrite.c>
  RewriteEngine On
  RewriteBase /
  RewriteRule ^index\.html$ - [L]
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteRule . /index.html [L]
</IfModule>
```

</CodeGroup>

## Troubleshooting

### Common Issues

#### Port Already in Use

```bash
# Error: Port 4200 is already in use
# Solution: Use a different port
ng serve --port 4201
```

#### Node Version Mismatch

```bash
# Error: The engine "node" is incompatible
# Solution: Use nvm to switch versions
nvm use 24
```

#### Package Installation Errors

```bash
# Clear npm cache and reinstall
rm -rf node_modules package-lock.json
npm cache clean --force
npm install
```

#### Build Memory Errors

```bash
# Increase Node.js memory limit
node --max-old-space-size=4096 node_modules/@angular/cli/bin/ng build
```

### Getting Help

If you encounter issues:

1. Check the browser console for errors (F12)
2. Review the terminal output for build errors
3. Verify all dependencies are installed: `npm list`
4. Ensure Node.js version: `node --version`
5. Check the Network tab for API errors
