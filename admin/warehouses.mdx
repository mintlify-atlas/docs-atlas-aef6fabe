## Overview

The Warehouse Management module enables multi-location inventory tracking. Define warehouse locations, configure stock distribution, and manage inventory across multiple physical sites for accurate stock control.

<Note>
  This module requires the **Bodegas** permission with `canView` enabled. Creating and editing warehouses requires `canCreate` and `canEdit` permissions.
</Note>

## Warehouse Management Workflow

<Steps>
  <Step title="Access Warehouse Module">
    Navigate to **Master > Warehouses** from the main menu. You'll see a table listing all warehouses with:
    - Name
    - Code
    - City
    - Address
    - Phone
    - Active status
  </Step>

  <Step title="Create New Warehouse">
    Click the **New Warehouse** button to open the creation dialog.
    
    **Required Fields:**
    - **Name**: Descriptive warehouse name
    - **Code**: Short identifier code
    
    **Optional Fields:**
    - **Address**: Physical location
    - **City**: City name
    - **Phone**: Contact number
    - **Active**: Toggle on/off (default: active)
    
    <Warning>
      Name and Code are mandatory. The system will show a warning if either is missing.
    </Warning>
  </Step>

  <Step title="Configure Warehouse Details">
    Fill in the warehouse information:
    
    - **Name**: Use descriptive names like "Main Warehouse - Medellín" or "Retail Store #5"
    - **Code**: Use consistent naming conventions (e.g., WH-001, STR-005)
    - **Address**: Full street address for shipping and receiving
    - **City**: City name for reporting and logistics
    - **Phone**: Direct contact for warehouse staff
    
    <Info>
      Warehouse codes should follow a consistent pattern across your organization for easy identification.
    </Info>
  </Step>

  <Step title="Save Warehouse">
    Click **Save** to create the warehouse. The system:
    
    1. Validates required fields (Name and Code)
    2. Creates the warehouse record
    3. Associates it with your company (companiaId)
    4. Sets active status to true by default
    5. Reloads the warehouse list
    
    The new warehouse is now available for inventory assignment.
  </Step>

  <Step title="Edit Existing Warehouse">
    Click the **Edit** icon on any warehouse row to modify:
    - Warehouse details
    - Contact information
    - Active status
    
    <Note>
      Editing a warehouse does not affect existing inventory allocations. Stock remains in place.
    </Note>
  </Step>

  <Step title="Deactivate Warehouse">
    Instead of deleting, you can deactivate a warehouse:
    
    1. Click the **Delete** icon
    2. System sets `active = false` (soft delete)
    3. Warehouse is hidden from active lists
    4. Historical data and inventory remain intact
    
    <Info>
      Deactivating preserves data integrity. You can reactivate warehouses later if needed.
    </Info>
  </Step>
</Steps>

## Validation Rules

### Name Field
- **Required**: Yes
- **Format**: Any text
- **Best Practice**: Use descriptive, unique names
- **Example**: "Main Distribution Center", "Retail Store Bogotá"

### Code Field
- **Required**: Yes
- **Format**: Alphanumeric, typically short (3-10 characters)
- **Best Practice**: Use consistent prefixes (WH-, STR-, DC-)
- **Example**: "WH-001", "STR-BGT", "DC-MED"

### Address Field
- **Required**: No
- **Format**: Free text
- **Best Practice**: Use complete addresses for logistics
- **Example**: "Calle 10 #43-21, Medellín, Antioquia"

### City Field
- **Required**: No
- **Format**: Text
- **Use**: Filtering, reporting, logistics planning

### Phone Field
- **Required**: No
- **Format**: Phone number (format not enforced)
- **Best Practice**: Include country/area codes
- **Example**: "+57 300 123 4567"

### Active Status
- **Default**: true
- **Type**: Boolean
- **Purpose**: Soft delete mechanism

filterWarehouses() {
  const term = this.searchTerm.toLowerCase();
  this.filteredWarehouses = this.warehouses.filter((b) =>
    b.name.toLowerCase().includes(term) ||
    b.code.toLowerCase().includes(term) ||

## Multi-Location Stock Tracking

The warehouse system enables:

1. **Stock Distribution**: View inventory across all locations
2. **Location-Specific Sales**: Track which warehouse fulfills orders
3. **Transfer Management**: Move inventory between warehouses
4. **Reorder Alerts**: Get low-stock notifications per warehouse

<Note>
  The POS system automatically deducts stock from the selected warehouse when processing sales.
</Note>

## API Integration

### Load Warehouses
```typescript
this.masterService.getWarehouses(this.companiaId).subscribe({
  next: (res: ApiResponse<Warehouse[]>) => {
    this.warehouses = res.data || [];
    this.filteredWarehouses = this.warehouses;
  }
});
```

### Create or Update Warehouse
```typescript
this.masterService.createWarehouse(this.newWarehouse).subscribe({
  next: (res) => {
    if (res.code === 0) {
      // Success
      this.showDialog = false;
      this.loadWarehouses();
    }
  }
});
```

### Deactivate Warehouse (Soft Delete)
```typescript
deleteWarehouse(warehouse: Warehouse) {
  const payload = { ...warehouse, active: false };
  this.masterService.deleteWarehouse(payload).subscribe({
    next: (res) => {
      if (res.code === 0) {
        this.loadWarehouses();
      }
    }
  });
}
```

<Info>
  The system uses soft deletion (setting `active = false`) to preserve historical inventory data.
</Info>

## Permission Requirements

The component implements role-based access control:

```typescript
private applyPermissions(): void {
  const moduleName = 'Bodegas';
  const permission = this.permissions.find((p) => 
    p.module === moduleName
  );
  
  this.canView = permission.canView;      // View warehouse list
  this.canCreate = permission.canCreate;  // Create new warehouse
  this.canEdit = permission.canEdit;      // Edit warehouse details
  this.canDelete = permission.canDelete;  // Deactivate warehouse
  this.canExport = permission.canExport;  // Export to Excel/CSV
}
```

UI buttons are conditionally rendered:

```html
<button *ngIf="canCreate" (click)="openNewDialog()">New Warehouse</button>
<button *ngIf="canEdit" (click)="editWarehouse(warehouse)">Edit</button>
<button *ngIf="canDelete" (click)="deleteWarehouse(warehouse)">Delete</button>
```

## Best Practices

<Info>
  **Consistent naming**: Use prefixes like WH-, STR-, DC- to quickly identify warehouse types.
</Info>

<Warning>
  **Don't delete warehouses with inventory**: Always deactivate instead. Deleting can orphan inventory records.
</Warning>

<Note>
  **Set up before adding products**: Create all warehouses before configuring product inventory to streamline setup.
</Note>

## Troubleshooting

### Warehouse Not Saving
- Verify Name and Code fields are filled
- Check that you have `canCreate` or `canEdit` permission
- Look for validation errors in browser console

### Warehouse Not Appearing in Product Form
- Ensure warehouse `active` status is true
- Check that warehouse belongs to the same `companiaId`
- Refresh the product management page

### Can't Delete Warehouse
- Verify you have `canDelete` permission
- Check if inventory is still assigned to this warehouse
- Review error message in toast notification

### Stock Not Updating
- Confirm sales are configured to deduct from correct warehouse
- Check product's `manageStock` setting is enabled
- Verify warehouse ID in stock allocation
