## Overview

The Category Management module enables product organization through hierarchical categorization. Create, edit, and manage product categories to improve inventory organization, facilitate reporting, and enhance the shopping experience.

<Note>
  This module requires the **Categorías** permission with `canView` enabled. Creating and editing categories requires `canCreate` and `canEdit` permissions.
</Note>

## Category Management Workflow

<Steps>
  <Step title="Access Categories Module">
    Navigate to **Master > Categories** from the main menu. You'll see a table displaying all categories with:
    - Category Name
    - Description
    - Active Status
    - Action buttons (Edit/Delete)
  </Step>

  <Step title="Create New Category">
    Click the **New Category** button to open the creation dialog.
    
    **Required Fields:**
    - **Name**: Category name (must not be empty)
    
    **Optional Fields:**
    - **Description**: Detailed category description
    - **Active**: Toggle category status (default: active)
    
    <Warning>
      Category name is mandatory. The system will show a warning toast if it's empty or contains only whitespace.
    </Warning>
  </Step>

  <Step title="Define Category Details">
    Fill in the category information:
    
    - **Name**: Use clear, descriptive names (e.g., "Electronics", "Dairy Products")
    - **Description**: Add context about what products belong in this category
    - **Active**: Keep enabled for categories in current use
    
    <Info>
      Good category names improve product findability and reporting clarity.
    </Info>
  </Step>

  <Step title="Save Category">
    Click **Save** to create the category. The system:
    
    1. Validates that the name field is not empty
    2. Trims whitespace from the name
    3. Creates the category record
    4. Associates it with your company (companiaId)
    5. Sets active status to true by default
    6. Reloads the category list
    
    The category is now available for product assignment.
  </Step>

  <Step title="Edit Existing Category">
    Click the **Edit** icon on any category row to modify:
    - Category name
    - Description
    - Active status
    
    <Note>
      Editing a category name updates it for all products already assigned to this category.
    </Note>
  </Step>

  <Step title="Deactivate Category">
    Instead of deleting, deactivate categories no longer in use:
    
    1. Click the **Delete** icon
    2. System sets `active = false` (soft delete)
    3. Category is hidden from active dropdown lists
    4. Products with this category retain the assignment
    
    <Info>
      Deactivating preserves historical data. Products keep their category for reporting purposes.
    </Info>
  </Step>
</Steps>

## Validation Rules

### Name Field
- **Required**: Yes
- **Format**: Any non-empty text
- **Validation**: Trimmed before saving (`name.trim()`)
- **Best Practice**: Use title case (e.g., "Electronic Accessories" not "electronic accessories")
- **Example**: "Beverages", "Frozen Foods", "Office Supplies"

### Description Field
- **Required**: No
- **Format**: Free text
- **Best Practice**: Explain category scope and typical products
- **Example**: "All beverages including soft drinks, juices, and bottled water"

### Active Status
- **Default**: true
- **Type**: Boolean
- **Purpose**: Soft delete mechanism to hide categories without losing data

## Search and Filter

The component provides real-time search across category fields:

```typescript
filterCategories() {
  const term = this.searchTerm.toLowerCase();
  this.filteredCategories = this.categories.filter((cat) =>
    cat.name.toLowerCase().includes(term) ||
    (cat.description?.toLowerCase().includes(term) ?? false)
  );
}
```

Type in the search box to filter categories by:
- **Name**: Full or partial category name
- **Description**: Keywords in the description

<Info>
  The search is case-insensitive and matches partial strings. Searching "elec" will find "Electronics".
</Info>

## API Integration

### Load Categories
```typescript
this.masterService.getCategories(this.companiaId).subscribe({
  next: (res: ApiResponse<Category[]>) => {
    this.categories = res.data || [];
    this.filteredCategories = this.categories;
  },
  error: () => {
    this.messageService.add({
      severity: 'error',
      summary: 'Error',
      detail: 'No se pudieron cargar las categorías.'
    });
  }
});
```

### Create or Update Category
```typescript
this.masterService.createCategory(this.newCategory).subscribe({
  next: (res) => {
    if (res.code === 0) {
      this.messageService.add({
        severity: 'success',
        summary: 'Éxito',
        detail: isEditing 
          ? 'Categoría actualizada correctamente.' 
          : 'Categoría creada correctamente.'
      });
      this.showDialog = false;
      this.loadCategories();
    }
  }
});
```

### Deactivate Category (Soft Delete)
```typescript
deleteCategory(cat: Category) {
  this.masterService.deleteCategory({ ...cat, active: false }).subscribe({
    next: (res) => {
      if (res.code === 0) {
        this.messageService.add({
          severity: 'info',
          summary: 'Desactivada',
          detail: `Categoría "${cat.name}" desactivada correctamente.`
        });
        this.loadCategories();
      }
    }
  });
}
```

<Info>
  The system uses soft deletion to preserve data integrity. Deactivated categories remain in the database but are hidden from dropdown lists.
</Info>

## Permission Requirements

The component implements role-based access control:

```typescript
private applyPermissions(): void {
  const moduleName = 'Categorías';
  const permission = this.permissions.find((p) => 
    p.module === moduleName
  );
  
  this.canView = permission.canView;      // View category list
  this.canCreate = permission.canCreate;  // Create new category
  this.canEdit = permission.canEdit;      // Edit category details
  this.canDelete = permission.canDelete;  // Deactivate category
  this.canExport = permission.canExport;  // Export to Excel/CSV
}
```

UI elements are conditionally rendered based on permissions:

```html
<button *ngIf="canCreate" (click)="openNewDialog()">New Category</button>
<button *ngIf="canEdit" (click)="editCategory(cat)">Edit</button>
<button *ngIf="canDelete" (click)="deleteCategory(cat)">Delete</button>
```

## Best Practices

<Info>
  **Use consistent naming**: Establish naming conventions before creating categories (e.g., all singular or all plural).
</Info>

<Warning>
  **Don't over-categorize**: Too many categories make product assignment confusing. Start with broad categories and add more as needed.
</Warning>

<Note>
  **Plan before implementation**: Sketch your category structure on paper before creating categories in the system.
</Note>

<Info>
  **Regular review**: Audit categories quarterly. Merge or deactivate unused categories to keep the system clean.
</Info>

## Troubleshooting

### Category Not Saving
- Ensure the Name field is not empty
- Remove leading/trailing spaces (system auto-trims)
- Verify you have `canCreate` or `canEdit` permission
- Check browser console for API errors

### Category Not Appearing in Product Dropdown
- Verify category `active` status is true
- Confirm category belongs to the same `companiaId`
- Refresh the product management page
- Check that category was saved successfully

### Can't Delete Category
- Verify you have `canDelete` permission
- Check if products are assigned to this category
- Review error message in toast notification
- Consider deactivating instead of deleting

### Duplicate Category Names
- System doesn't enforce unique names (by design)
- Manually check for duplicates before creating
- Use search to find similar category names
- Consider adding context to similar names (e.g., "Beverages - Hot", "Beverages - Cold")
