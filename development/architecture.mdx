## Architecture Pattern

### Standalone Components

The application uses Angular 19's **standalone components** architecture, eliminating the need for NgModules:

```typescript
import { Component, OnInit } from '@angular/core';
import { CommonModule } from '@angular/common';
import { FormsModule } from '@angular/forms';

// PrimeNG Imports
import { ButtonModule } from 'primeng/button';
import { TableModule } from 'primeng/table';

@Component({
    selector: 'app-agents',
    standalone: true,
    imports: [
        CommonModule, 
        FormsModule, 
        TableModule, 
        ButtonModule
    ],
    templateUrl: './agents.component.html',
    styleUrl: './agents.component.scss'
})
export class AgentsComponent implements OnInit {
    // Component logic
}
```

### Key Benefits

- **Simplified Module Management**: No need for NgModules
- **Better Tree-Shaking**: Unused imports are automatically removed
- **Improved Performance**: Lazy loading at component level
- **Enhanced Developer Experience**: Clearer dependencies

## Service Layer

### Service Pattern

Services use dependency injection and RxJS observables:

```typescript
import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable({
    providedIn: 'root'
})
export class MasterService {
    private apiUrl = 'https://api.example.com';
    
    constructor(private http: HttpClient) {}
    
    getUsers(companiaId: number): Observable<any> {
        return this.http.get(`${this.apiUrl}/users/${companiaId}`);
    }
    
    createUser(user: User): Observable<any> {
        return this.http.post(`${this.apiUrl}/users`, user);
    }
}
```

### Service Injection

Services are provided at root level for singleton behavior:

```typescript
@Injectable({
    providedIn: 'root'  // Single instance across app
})
```

### Permission-Based UI

Components implement role-based access control:

```typescript
private applyPermissions(): void {
    const moduleName = 'Agentes';
    const permission = this.permissions.find(p => p.module === moduleName);
    
    if (!permission) {
        this.resetPermissions();
        return;
    }
    
    this.canView = permission.canView;
    this.canCreate = permission.canCreate;
    this.canEdit = permission.canEdit;
    this.canDelete = permission.canDelete;
    this.canExport = permission.canExport;
}
```

## Best Practices

<Tip>
**Component Organization**
- Keep components focused on a single responsibility
- Use services for business logic and data access
- Implement proper error handling and loading states
- Apply permission checks for role-based access
</Tip>

<Warning>
**Common Pitfalls**
- Don't forget to unsubscribe from observables
- Avoid logic in templates - move to component class
- Don't manipulate DOM directly - use Angular directives
- Keep component selector prefix consistent (`app-`)
</Warning>
