## Overview

The application uses a hybrid styling approach combining:

- **Tailwind CSS** - Utility-first CSS framework
- **SCSS** - Component-specific styles
- **PrimeNG Themes** - UI component theming
- **CSS Variables** - Dynamic theming support

## Tailwind CSS Configuration

The application uses Tailwind CSS v3.4.17 with the PrimeUI plugin.

### Configuration File

**Location**: `tailwind.config.js`

<CodeGroup>
```javascript tailwind.config.js
import PrimeUI from 'tailwindcss-primeui';

export default {
    darkMode: ['selector', '[class="app-dark"]'],
    content: [
        './src/**/*.{html,ts,scss,css}',
        './index.html'
    ],
    plugins: [PrimeUI],
    theme: {
        extend: {
            colors: {
                background: 'hsl(var(--background))',
                foreground: 'hsl(var(--foreground))',
                primary: {
                    DEFAULT: 'hsl(var(--primary))',
                    foreground: 'hsl(var(--primary-foreground))'
                },
                secondary: {
                    DEFAULT: 'hsl(var(--secondary))',
                    foreground: 'hsl(var(--secondary-foreground))'
                },
                // ... more colors
            },
            borderRadius: {
                lg: 'var(--radius)',
                md: 'calc(var(--radius) - 2px)',
                sm: 'calc(var(--radius) - 4px)'
            }
        }
    }
};
```
</CodeGroup>

### Key Features

- **Dark Mode**: Supports selector-based dark mode switching
- **Content Paths**: Scans all HTML, TypeScript, and SCSS files
- **PrimeUI Plugin**: Integrates Tailwind with PrimeNG components
- **Custom Colors**: HSL-based color system using CSS variables

## CSS Variables

The application uses HSL-based CSS variables for theming:

### Light Theme

```scss
:root {
    --background: 0 0% 100%;           /* White */
    --foreground: 0 0% 3.9%;           /* Almost black */
    --primary: 0 0% 9%;                 /* Dark gray */
    --primary-foreground: 0 0% 98%;     /* Off white */
    --secondary: 0 0% 96.1%;            /* Light gray */
    --border: 0 0% 89.8%;               /* Border gray */
    --radius: 0.5rem;                   /* Border radius */
}
```

### Dark Theme

```scss
.dark {
    --background: 0 0% 3.9%;            /* Dark background */
    --foreground: 0 0% 98%;             /* Light text */
    --primary: 0 0% 98%;                /* Light primary */
    --primary-foreground: 0 0% 9%;      /* Dark text */
    --secondary: 0 0% 14.9%;            /* Dark gray */
    --border: 0 0% 14.9%;               /* Dark border */
}
```

### Using CSS Variables

Access variables through Tailwind classes:

```html
<div class="bg-background text-foreground">
    <button class="bg-primary text-primary-foreground">
        Click me
    </button>
</div>
```

## Component Styles (SCSS)

Each component can have its own SCSS file:

### Component SCSS Pattern

```scss
// agents.component.scss
:host {
    display: block;
}

.card {
    @apply bg-white rounded-lg shadow-sm p-6;
    
    &:hover {
        @apply shadow-md;
    }
}

.status-badge {
    @apply inline-flex items-center px-3 py-1 rounded-full text-sm font-medium;
    
    &.active {
        @apply bg-green-100 text-green-800;
    }
    
    &.inactive {
        @apply bg-red-100 text-red-800;
    }
}

// Custom scrollbar
.custom-scrollbar::-webkit-scrollbar {
    width: 5px;
}

.custom-scrollbar::-webkit-scrollbar-thumb {
    background: #e2e8f0;
    border-radius: 10px;
}
```

### Using @apply

The `@apply` directive lets you use Tailwind classes in SCSS:

```scss
.button-primary {
    @apply bg-blue-600 text-white px-4 py-2 rounded-lg;
    @apply hover:bg-blue-700 transition-colors;
    @apply focus:ring-2 focus:ring-blue-500 focus:ring-offset-2;
}
```

## PrimeNG Theming

The application uses PrimeNG 19 with the new theming system:

### PrimeNG Imports

```typescript
import { ButtonModule } from 'primeng/button';
import { TableModule } from 'primeng/table';
import { DialogModule } from 'primeng/dialog';
import { DropdownModule } from 'primeng/dropdown';
```

### Styling PrimeNG Components

Override PrimeNG styles using `:host ::ng-deep`:

```scss
:host ::ng-deep {
    .p-button {
        @apply rounded-lg font-semibold;
    }
    
    .p-datatable {
        .p-datatable-header {
            @apply bg-slate-50 border-b border-slate-200;
        }
        
        .p-datatable-tbody > tr:hover {
            @apply bg-slate-50;
        }
    }
}
```

## Responsive Design

### Breakpoints

Tailwind's default breakpoints:

| Breakpoint | Min Width | Usage |
|------------|-----------|-------|
| `sm` | 640px | Small devices |
| `md` | 768px | Tablets |
| `lg` | 1024px | Laptops |
| `xl` | 1280px | Desktops |

```javascript
keyframes: {

        from: { height: '0' },
        to: { height: 'var(--radix-accordion-content-height)' }
    }
},
animation: {
    'accordion-down': 'accordion-down 0.2s ease-out'
}
```

## Best Practices

<Tip>
**Styling Guidelines**
- Use Tailwind utilities for common patterns
- Use SCSS for complex component-specific styles
- Leverage CSS variables for theming
- Keep styles scoped to components
- Use `@apply` sparingly - prefer utility classes in templates
</Tip>

<Warning>
**Common Mistakes**
- Don't use inline styles - use Tailwind classes
- Don't override PrimeNG styles globally - scope to components
- Don't forget to test both light and dark modes
- Avoid using `!important` unless absolutely necessary
</Warning>
