## ESLint Configuration

The project uses ESLint v9.14.0 with Angular-specific rules.

### Configuration File

**Location**: `eslint.config.js`

<CodeGroup>
```javascript eslint.config.js
export default {
    root: true,
    ignorePatterns: ['**/dist/**'],
    plugins: ['prettier'],
    extends: ['prettier'],
    rules: {
        'padding-line-between-statements': [
            'error',
            { blankLine: 'always', prev: ['const', 'let', 'var'], next: '*' },
            { blankLine: 'any', prev: ['const', 'let', 'var'], next: ['const', 'let', 'var'] },
            { blankLine: 'always', prev: '*', next: 'return' },
            { blankLine: 'always', prev: 'block', next: '*' },
            { blankLine: 'always', prev: '*', next: 'block' }
        ]
    },
    overrides: [
        {
            files: ['*.ts'],
            extends: [
                'eslint:recommended',
                'plugin:@typescript-eslint/recommended',
                'plugin:@angular-eslint/recommended',
                'prettier'
            ],
            rules: {
                '@angular-eslint/component-selector': [
                    'error',
                    {
                        type: 'element',
                        prefix: 'p',
                        style: 'kebab-case'
                    }
                ],
                '@angular-eslint/directive-selector': [
                    'error',
                    {
                        type: 'attribute',
                        prefix: 'p',
                        style: 'camelCase'
                    }
                ],
                '@typescript-eslint/no-explicit-any': 'off',
                'arrow-body-style': ['error', 'as-needed'],
                'prefer-const': 0
            }
        }
    ]
};
```
</CodeGroup>

### Key ESLint Rules

**Component Selectors**:
- **Prefix**: `p` (e.g., `<p-button>`, `<p-table>`)
- **Style**: `kebab-case`
- **Type**: Element selector

**Directive Selectors**:
- **Prefix**: `p`
- **Style**: `camelCase`
- **Type**: Attribute selector

**Code Style**:
- **Blank Lines**: Required before returns, blocks, and after variable declarations
- **Arrow Functions**: Use concise body when possible
- **No Explicit Any**: Disabled (allowed)
- **Prefer Const**: Disabled (let is allowed)

### Formatting Rules

| Rule | Value | Description |
|------|-------|-------------|
| **Tab Width** | 4 spaces | Indentation size |
| **Quotes** | Single | Use single quotes for strings |
| **Semicolons** | Always | Require semicolons |
| **Print Width** | 250 | Maximum line length |
| **Trailing Comma** | None | No trailing commas |
| **Tabs** | Spaces | Use spaces, not tabs |

### Running Prettier

Format all code:

```bash
npm run format
```

This runs:
```bash
prettier --write "**/*.{js,mjs,ts,mts,d.ts,html}" --cache
```

<Tip>
Prettier caches results for faster subsequent runs. The cache is automatically managed.
</Tip>

### Type Annotations

Always specify types explicitly:

<CodeGroup>
```typescript Good
// Explicit types
const users: User[] = [];
const isLoading: boolean = false;
const selectedId: number | null = null;

function getUser(id: number): Observable<User> {
    return this.http.get<User>(`/api/users/${id}`);
}
```

```typescript Bad
// Implicit any types
const users = [];
let isLoading;
const selectedId = null;

function getUser(id) {
    return this.http.get(`/api/users/${id}`);
}
```
</CodeGroup>

### Interfaces and Types

Define interfaces for all data structures:

```typescript
// User interface
export interface User {
```

## Angular Code Standards

### Component Structure

Follow a consistent component structure:

```typescript
import { Component, OnInit } from '@angular/core';
import { CommonModule } from '@angular/common';

// 1. Angular imports
// 2. Third-party imports
// 3. Application imports

@Component({
    selector: 'app-example',
    standalone: true,
    imports: [CommonModule],
    templateUrl: './example.component.html',
    styleUrl: './example.component.scss'
})
export class ExampleComponent implements OnInit {
    // 1. Public static fields
    // 2. Private static fields
    // 3. Public instance fields
    // 4. Private instance fields
    
    // Public properties
    users: User[] = [];
    isLoading: boolean = false;
    
    // Private properties
    private companiaId: number = 1;
    
    // 5. Constructor
    constructor(
        private userService: UserService,
        private messageService: MessageService
    ) {}
    
    // 6. Lifecycle hooks
    ngOnInit(): void {
        this.loadUsers();
    }
    
    // 7. Public methods
    public loadUsers(): void {
        this.isLoading = true;
        this.userService.getUsers().subscribe({
            next: (response) => this.handleSuccess(response),
            error: (error) => this.handleError(error)
        });
    }
    
    // 8. Private methods
    private handleSuccess(response: ApiResponse<User[]>): void {
        this.users = response.data || [];
        this.isLoading = false;
    }
    
    private handleError(error: any): void {
        this.isLoading = false;
        this.messageService.add({
            severity: 'error',
            summary: 'Error',
            detail: 'Failed to load users'
        });
    }
}
```

### Member Ordering

ESLint enforces member ordering:

```typescript
export class ExampleComponent {

    // 2. Static fields

    // 4. Constructor
    constructor() {}
    
    // 5. Public instance methods
    public loadData(): void {}
    
    // 6. Private instance methods
    private handleError(): void {}
}
```

## RxJS Standards

### Observable Subscriptions

```typescript
import { Subscription } from 'rxjs';

export class ExampleComponent implements OnInit, OnDestroy {
    private subscriptions = new Subscription();
    
    ngOnInit(): void {
        // Add subscriptions
        this.subscriptions.add(
            this.userService.getUsers().subscribe({
                next: (users) => this.users = users,
                error: (error) => this.handleError(error)
            })
        );
    }
    
    ngOnDestroy(): void {
        // Unsubscribe all
        this.subscriptions.unsubscribe();
    }
}
```

### Observable Error Handling

```typescript
this.userService.getUsers().subscribe({
    next: (response) => {
        this.users = response.data || [];
        this.isLoading = false;
    },
    error: (error) => {
        console.error('Failed to load users:', error);
        this.isLoading = false;
        this.showError('Failed to load users');
    }
});
```

## Permissions Pattern

Implement permission checks consistently:

```typescript
export class ExampleComponent implements OnInit {
    // Permission flags
    canView = false;
    canCreate = false;
    canEdit = false;
    canDelete = false;
    canExport = false;
    
    private permissions: Permission[] = [];
    
    ngOnInit(): void {
        this.loadPermissions();
    }
    
    private loadPermissions(): void {
        const session = this.authService.getSession();
        
        if (!session) {
            this.resetPermissions();
            return;
        }
        
        this.loginService.getPermissions(session.userId, session.companiaId)
            .subscribe({
                next: (permissions) => {
                    this.permissions = permissions.data ?? [];
                    this.applyPermissions();
                },
                error: () => this.resetPermissions()
            });
    }
    
    private applyPermissions(): void {
        const moduleName = 'ModuleName';
        const permission = this.permissions.find(p => p.module === moduleName);
        
        if (!permission) {
            this.resetPermissions();
            return;
        }
        
        this.canView = permission.canView;
        this.canCreate = permission.canCreate;
        this.canEdit = permission.canEdit;
        this.canDelete = permission.canDelete;
        this.canExport = permission.canExport;
    }
    
    private resetPermissions(): void {
        this.canView = false;
        this.canCreate = false;
        this.canEdit = false;
        this.canDelete = false;
        this.canExport = false;
    }
}
```

## Git Commit Standards

### Commit Message Format

```bash
# Format: <type>: <subject>

feat: add user management module
fix: resolve navigation issue in sidebar
style: format code with prettier
refactor: simplify user service logic
docs: update API documentation
test: add unit tests for auth service
```

### Commit Types

| Type | Description |
|------|-------------|
| `feat` | New feature |
| `fix` | Bug fix |
| `docs` | Documentation only |
| `style` | Code style changes (formatting) |
| `refactor` | Code refactoring |
| `test` | Adding or updating tests |
| `chore` | Build process or auxiliary tool changes |

## Code Review Checklist

<Steps>
  <Step title="Code Quality">
    - Code follows ESLint rules
    - All files formatted with Prettier
    - No console.log statements (use proper logging)
    - No commented-out code
  </Step>

  <Step title="TypeScript">
    - All types explicitly defined
    - No use of `any` type (unless necessary)
    - Interfaces defined for data structures
    - Proper null/undefined handling
  </Step>

  <Step title="Angular">
    - Component follows single responsibility
    - Proper lifecycle hook usage
    - Observables properly unsubscribed
    - Standalone component imports correct
  </Step>

  <Step title="Testing">
    - Unit tests included
    - Tests pass successfully
    - Code coverage acceptable
  </Step>
</Steps>

## Best Practices

<Tip>
**Code Quality Tips**
- Run `npm run format` before committing
- Fix all ESLint warnings and errors
- Write self-documenting code with clear variable names
- Add JSDoc comments for public APIs
- Keep functions small and focused
</Tip>

<Warning>
**Avoid These Mistakes**
- Don't disable ESLint rules without good reason
- Don't use `any` type as a shortcut
- Don't forget to unsubscribe from observables
- Don't mix tabs and spaces
- Don't commit commented-out code
</Warning>
