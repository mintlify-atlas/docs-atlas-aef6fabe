# Invoice Management

The Invoice Management module provides comprehensive tools for viewing, searching, filtering, and managing all sales invoices generated through the POS system.

## Overview

Built with Angular 19 and PrimeNG, this module offers advanced invoice querying, detailed views with line items, PDF generation for printing and downloads, and administrative functions like cancellation.

## Filtering Invoices

### Local Filters

Client-side filtering for fast responses:

```typescript
applyLocalFilters() {
    this.filteredInvoices = this.invoices.filter((inv) => {
        const matchText = !this.filterClient ||
            inv.client_Name?.toLowerCase().includes(
                this.filterClient.toLowerCase()
            ) ||
            inv.invoice_Number?.toString().includes(this.filterClient);
        
        const matchStatus = !this.filterStatus || 
            inv.invoice_Status === this.filterStatus;
        
        return matchText && matchStatus;
    });
}
```
**Source**: `invoices.component.ts:197-205`

This filters by:
- **Client name** (case-insensitive)
- **Invoice number** (partial match)
- **Status** (exact match)

## Invoice Statistics

Real-time calculation of totals by status:

```typescript
getStats() {
    const stats = {
        totalAprobada: 0,
        countAprobada: 0,
        totalAnulada: 0,
        countAnulada: 0,
        totalRechazada: 0,
        countRechazada: 0
    };
    
    // Uses filteredInvoices so stats change when filtering
    this.filteredInvoices.forEach((inv) => {
        const total = inv.invoice_Total || 0;
        const status = inv.invoice_Status?.toUpperCase();
        
        if (status === 'APROBADA') {
            stats.totalAprobada += total;
            stats.countAprobada++;
        } else if (status === 'ANULADA') {
            stats.totalAnulada += total;
            stats.countAnulada++;
        } else if (status === 'RECHAZADA') {
            stats.totalRechazada += total;
            stats.countRechazada++;
        }
    });
    
    return stats;
}
```
**Source**: `invoices.component.ts:207-235`

<Note>
  Statistics are calculated from `filteredInvoices`, so they update dynamically when filters are applied.
</Note>

## PDF Generation

The module supports two PDF formats:

### POS Receipt Format

Optimized for thermal printers:

```typescript
handlePrint(): void {
    let detailDiscount = this.selectedInvoice?.items?.reduce(
        (sum, item) => sum + (item.discount_Value || 0), 0
    ) || 0;
    
    this.invoicePdfService.generatePOS({
        invoiceNumber: this.selectedInvoice?.invoice_Number || '',
        date: this.selectedInvoice?.created_At || new Date(),
        customer: this.selectedInvoice?.customer || null,
        items: this.selectedInvoice?.items || [],
        subtotal: this.selectedInvoice?.subtotal || 0,
        detailDiscount: this.selectedInvoice?.items?.reduce(
            (acc, item) => acc + (item.discount_Value ?? 0), 0
        ) ?? 0,
        generalDiscount: this.selectedInvoice?.descuento_General || 0,
        grossSubtotal: this.selectedInvoice?.subtotal_Bruto || 0,
        discount: detailDiscount + 
            (((this.selectedInvoice?.subtotal || 0) - detailDiscount) * 
            (this.selectedInvoice?.descuento_General || 0)) / 100,
        total: this.selectedInvoice?.invoice_Total || 0,
        deliveryInfo: this.selectedInvoice?.delivery,
        paymentMethods: this.selectedInvoice?.payments || [],
        session: this.selectedCompany
    });
}
```

## Cancelling Invoices

Invoices can be cancelled with confirmation:

```typescript
cancelInvoice() {
    let companyId = 1;
    
    this.confirmationService.confirm({
        message: `¿Está seguro de anular la factura ${this.selectedInvoice?.invoice_Number}?`,
        header: 'Confirmación',
        icon: 'pi pi-exclamation-triangle',
        accept: () => {
            this.posService.cancelInvoice(
                this.selectedInvoice?.id,
                companyId
            ).subscribe({
                next: (res) => {
                    this.loadInvoices();
                    this.messageService.add({
                        severity: 'success',
                        summary: 'Éxito',
                        detail: `Factura ${this.selectedInvoice?.invoice_Number} anulada correctamente`
                    });
                    this.loadInvoices();
                },
                error: (err) => {
                    this.messageService.add({
                        severity: 'error',
                        summary: 'Error',
                        detail: `No se pudo anular la factura ${this.selectedInvoice?.invoice_Number}`
                    });
                }
            });
        },
        reject: () => {
            this.messageService.add({
                severity: 'info',
                summary: 'Cancelado',
                detail: 'Acción de anular cancelada'
            });
        }
    });
}
```
**Source**: `invoices.component.ts:145-181`

<Warning>
  Cancelling an invoice is permanent. The status changes to "ANULADA" and the invoice is excluded from financial reports. Ensure you have proper permissions before cancelling.
</Warning>

## Discount Calculation

The module includes a helper to calculate total discounts:

```typescript
getDiscounts() {
    const general = this.selectedInvoice?.descuento_General ?? 0;
    const itemsDiscount = this.selectedInvoice?.items?.reduce(
        (acc, item) => acc + (item.discount_Value ?? 0), 0
    ) ?? 0;
    
    return general + itemsDiscount;
}
```
**Source**: `invoices.component.ts:321-327`

This combines:
- **Item-level discounts**: Sum of all `discount_Value` from line items
- **General discount**: Invoice-level discount percentage applied

private applyPermissions(): void {

## Best Practices

<Steps>
  <Step title="Use Date Ranges Wisely">
    Loading large date ranges can be slow. Default to today or last 7 days for better performance.
  </Step>
  
  <Step title="Filter Before Exporting">
    Apply filters to narrow down results before generating reports or exports.
  </Step>
  
  <Step title="Verify Before Cancelling">
    Always verify the invoice number and customer before cancelling. Cancellation is permanent.
  </Step>
  
  <Step title="Generate PDFs On-Demand">
    PDF generation can be resource-intensive. Only generate when needed rather than pre-generating.
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="No invoices loading">
    - Verify date range is correct
    - Check that company ID matches your session
    - Ensure you have `canView` permission for Facturas module
    - Check browser console for API errors
  </Accordion>
  
  <Accordion title="PDF not generating">
    - Verify the invoice has all required data (items, customer, etc.)
    - Check browser console for pdfMake errors
    - Ensure printer settings are configured in company settings
  </Accordion>
  
  <Accordion title="Cannot cancel invoice">
    - Verify you have `canDelete` permission
    - Some invoices may be locked after a certain period
    - Check if invoice is already cancelled (status: ANULADA)
  </Accordion>
  
  <Accordion title="Wrong totals displayed">
    The system calculates discounts as:
    1. Item-level discounts (from each product)
    2. General discount (percentage on subtotal)
    
    Verify both are being included in the total calculation.
  </Accordion>
</AccordionGroup>
