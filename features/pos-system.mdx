# POS System

The POS (Point of Sale) system is the core feature of the application, providing a comprehensive solution for managing sales transactions, inventory, and customer interactions in real-time.

## Overview

Built with Angular 19 and PrimeNG, the POS system offers a modern, responsive interface optimized for both desktop and touch-screen devices. It includes real-time stock validation, multiple payment methods, and seamless invoice generation.

## Stock Management

### Real-Time Stock Reservation

The POS system implements sophisticated stock management to prevent overselling:

<Steps>
  <Step title="Reserve Stock">
    When a product is added to the cart, stock is immediately reserved via the `reserveStock()` method:
    
    ```typescript
    reserveStock(item: CartItem, cantidad: number, existe: boolean) {
        this.posService
            .validateAndReserveStock({
                action: 'RESERVAR',
                productId: item.id,
                quantity: cantidad,
                companiaId: this.companiaId,
                userId: this.userId,
                facturaTempId: this.activeInvoice!.tempId
            })
            .subscribe({
                next: (res) => {
                    if (res.data.success != 1) {
                        // Rollback on insufficient stock
                        item.quantity = item.prevQuantity;
                        this.showToast('warn', 'Stock insuficiente', 
                            `Solo hay ${res.data.stockDisponible} unidades más disponibles`);
                    } else {
                        item.prevQuantity = item.quantity;
                        this.updateTotals(this.activeInvoice!);
                    }
                }
            });
    }
    ```
    **Source**: `pos.component.ts:337-374`
  </Step>
  
  <Step title="Release Stock">
    When quantities are reduced or items removed, stock is released back to inventory:
    
    ```typescript
    releaseStock(item: CartItem, cantidad: number) {
        this.posService
            .validateAndReserveStock({
                action: 'RESTAR',
                productId: item.id,
                quantity: cantidad,
                companiaId: this.companiaId,
                userId: this.userId,
                facturaTempId: this.activeInvoice!.tempId
            })
            .subscribe({
                next: () => {
                    item.prevQuantity = item.quantity;
                    if (item.quantity === 0) {
                        this.activeInvoice!.items = 
                            this.activeInvoice!.items.filter((i) => i !== item);
                    }
                    this.updateTotals(this.activeInvoice!);
                }
            });
    }
    ```
    **Source**: `pos.component.ts:376-402`
  </Step>
  
  <Step title="Automatic Cleanup">
    If an invoice is cleared or cancelled, all reserved stock is released:
    
    ```typescript
    clearInvoiceAll() {
        this.posService.cancelInvoiceAll(this.activeInvoice!.tempId, 1).subscribe({
            next: () => {
                this.activeInvoice!.tempId = uuidv4();
                this.showToast('success', 'Stock liberado', 
                    'Se ha liberado el stock reservado de la factura.');
            }
        });
    }
    ```
    **Source**: `pos.component.ts:404-415`
  </Step>
</Steps>

<Warning>
  Stock reservations are tied to a temporary invoice ID (`tempId`) using UUID v4. This ensures proper tracking and cleanup of reserved inventory.
</Warning>

## Total Calculation

The system calculates totals with proper discount and tax handling:

```typescript
updateTotals(invoice: Invoice): void {
    let grossSubtotal = 0;
    let totalDiscountDetail = 0;
    
    // Calculate per-item totals
    for (const item of invoice.items) {
        const discountPercent = item.discount || 0;
        const discountValue = (item.price * item.quantity * discountPercent) / 100;
        const itemSubtotal = item.price * item.quantity;
        const itemTotal = itemSubtotal - discountValue;
        
        item.discountValue = discountValue;
        item.subtotal = itemSubtotal;
        item.total = itemTotal;
        
        grossSubtotal += itemSubtotal;
        totalDiscountDetail += discountValue;
    }
    
    // Apply general discount
    const generalDiscountAmount = 
        (grossSubtotal - totalDiscountDetail) * (invoice.generalDiscount / 100);
    
    // Calculate VAT on net amount
    const netSubtotal = grossSubtotal - totalDiscountDetail - generalDiscountAmount;
    const ivaPercentage = this.ivaRate / 100;
    const totalVat = netSubtotal * ivaPercentage;
    
    // Final totals
    invoice.grossSubtotal = grossSubtotal;
    invoice.subtotal = netSubtotal;
    invoice.detailDiscount = totalDiscountDetail;
    invoice.totalVat = totalVat;
    invoice.total = netSubtotal + totalVat;
}
```
**Source**: `pos.component.ts:418-456`

<Info>
  The VAT rate is configurable and stored in the `ivaRate` property (line 74).
</Info>

### Save Invoice

```typescript
saveInvoice(invoice: Invoice) {
    const payload = {
        companiaId: this.companiaId,
        userId: this.userId,
        clientId: invoice.customer?.id || null,
        clientName: invoice.customer?.businessName || 
            `${invoice.customer?.firstName ?? ''} ${invoice.customer?.lastName ?? ''}`.trim() || 
            'Consumidor Final',
        subtotal: invoice.subtotal || 0,
        grossSubtotal: invoice.grossSubtotal || 0,
        generalDiscount: invoice.generalDiscount || 0,
        detailDiscount: invoice.detailDiscount || 0,
        totalVat: invoice.totalVat || 0,
        totalInvoice: invoice.total || 0,
        items: JSON.stringify(invoice.items.map(item => ({
            id: item.id,
            warehouseId: item.warehouseId ?? 1,
            quantity: item.quantity,
            price: item.price,
            discount: item.discount ?? 0,
            subtotal: item.subtotal,
            total: item.total
        }))),
        payments: JSON.stringify(invoice.paymentMethods),
        delivery: invoice.deliveryInfo ? JSON.stringify(invoice.deliveryInfo) : null
    };
    
    this.posService.createInvoice(payload).subscribe({
        next: (res) => {
            if (res.code === 0) {
                this.showToast('success', 'Venta completada', 
                    `Factura ${res.data[0].invoice_Number} registrada.`);
                this.showInvoiceDialog = true;
            }
        }
    });
}
```

## Best Practices

<Steps>
  <Step title="Always Validate Stock">
    The system automatically validates stock availability before allowing additions to the cart. Never bypass this validation.
  </Step>
  
  <Step title="Use Keyboard Shortcuts">
    For faster operation, train cashiers to use function keys (F2-F5) instead of mouse clicks.
  </Step>
  
  <Step title="Monitor Active Invoices">
    Keep track of how many invoices are pending to avoid confusion during busy periods.
  </Step>
  
  <Step title="Verify Payments">
    The checkout button only enables when payment equals total (within 0.01 tolerance).
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Stock shows as insufficient when it shouldn't be">
    Check if there are other pending invoices with reserved stock. Reserved stock is tied to the invoice's `tempId` and won't be released until the invoice is completed or cancelled.
  </Accordion>
  
  <Accordion title="Barcode scanner not working">
    Ensure the barcode input field has focus. The system automatically focuses it after operations, but if interrupted, click the barcode input field or press Tab to focus it.
  </Accordion>
  
  <Accordion title="Payment total doesn't match">
    The system uses a 0.01 tolerance for floating-point precision. Verify all payment amounts and ensure the sum equals the invoice total within this margin.
  </Accordion>
</AccordionGroup>
